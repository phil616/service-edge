# service-edge 项目指南

云梦镜像边缘服务网络（service-edge）——基于 FRP 的内网穿透管控系统。本指南说明如何构建、测试、部署本项目，以及部署运维中需要注意的事项。

设计依据见 [`design.md`](./design.md)，速览见 [`README.md`](./README.md)。

---

## 一、项目概述

| 组件 | 位置 | 说明 |
|---|---|---|
| 控制面后端 | `cmd/server` | API、业务逻辑、PKI、配置下发，单二进制（内嵌前端） |
| 控制面前端 | `web/` | React 管理控制台，构建后内嵌进后端二进制 |
| Agent | `cmd/agent` | 部署在 frps/frpc 主机，管理 frp 进程、上报状态、拉取配置 |

一次通信链路：

```
浏览器 ──(JWT)──▶ 控制面后端 ◀──(X-Agent-Token + long-polling)── Agent ──▶ frps/frpc 进程
```

技术栈：Go 1.22+ / Gin / GORM / SQLite（纯 Go，无 CGO）；React 18 + TypeScript + Vite + Ant Design。

## 二、环境要求

### 构建环境

| 工具 | 版本 | 用途 |
|---|---|---|
| Go | 1.22+（已用 1.26 验证） | 构建后端与 Agent |
| Node.js | 18+（已用 24 验证） | 构建前端 |
| npm | 9+ | 前端依赖管理 |
| make | 任意 | 构建快捷命令（可选） |
| git | 任意 | 版本管理、版本号注入 |

> **无需 CGO / gcc**：SQLite 驱动使用纯 Go 的 `glebarez/sqlite`，可直接交叉编译，`CGO_ENABLED=0` 也能构建。

### 运行环境

- **控制面**：任意 Linux 服务器，需持有 CA 证书与私钥。
- **Agent 主机**：Linux（amd64 / arm64），需 `systemd`、`curl`、`tar`，并以 root 安装。

## 四、如何构建

### 4.1 使用 Makefile（推荐）

```bash
make web          # 1. 构建前端 → 输出到 internal/web/dist
make server       # 2. 构建后端 → bin/service-edge（已内嵌前端）
make agent        # 3. 构建 Agent → bin/agent
make agents       # 4. 交叉编译 Agent（linux amd64 + arm64）
make all          # = server + agent + web
```

> **顺序很重要**：`make server` 通过 `go:embed` 把 `internal/web/dist` 打进二进制。若想让控制台带最新前端，必须先 `make web` 再 `make server`。新克隆仓库时 `internal/web/dist/` 仅含占位文件，直接构建后端可成功编译，但控制台页面为空，需先构建前端。

### 4.2 不使用 Makefile

```bash
# 前端
cd web && npm install && npm run build && cd ..
# 后端 + Agent
go build -o bin/service-edge ./cmd/server
go build -o bin/agent ./cmd/agent
# 交叉编译 Agent
GOOS=linux GOARCH=arm64 go build -o bin/agent_linux_arm64 ./cmd/agent
```

### 4.3 版本号注入

Makefile 通过 `-ldflags "-X main.version=..."` 注入 git 版本号；`./bin/agent --version` 可查看。

## 五、配置说明（config.yaml）

复制 `config.example.yaml` 为 `config.yaml` 后按需修改。各字段含义：

```yaml
server:
  listen: "0.0.0.0:8443"                 # 监听地址
  external_url: "https://edge-api.dreamreflex.com"  # 对外地址，注入到安装脚本与下载 URL

database:
  path: "/var/lib/service-edge/data.db"  # SQLite 文件路径，目录会自动创建

# Agent ↔ 控制面 API 的全局认证 token（≠ frpc↔frps 的 frp token），建议 64 位随机串
agent_api_token: "..."
# 用户登录 JWT 签名密钥，建议 64 位随机串
jwt_secret: "..."

pki:
  ca_cert: "/etc/service-edge/ca.crt"    # CA 证书（启动时强校验）
  ca_key:  "/etc/service-edge/ca.key"    # CA 私钥

frp_release:
  base_url: "https://github.com/fatedier/frp/releases/download"
  default_version: "v0.61.1"             # 新建节点的默认 frp 版本（可在表单覆盖）

install_script_base: "https://edge-api.dreamreflex.com/install"  # 安装脚本下载基址
agent_download_base: "https://edge-api.dreamreflex.com/download/agent"  # Agent 二进制下载基址（自动追加 _linux_amd64）

enrollment_token_ttl: "15m"              # 一次性安装 token 有效期

cors:
  allowed_origins:
    - "https://edge.dreamreflex.com"     # 前端域名白名单（严格匹配）

logging:
  level: "info"
  path: "/var/log/service-edge/server.log"

bootstrap_admin:                          # 首次启动且无任何用户时创建
  username: "admin"
  password: "admin12345"                  # 生产环境务必修改
```

生成随机密钥：`openssl rand -hex 32`。

## 六、本地开发与运行

### 6.1 一键起后端（含前端）

```bash
# 1. 生成开发用自签 CA（输出到 dev/ca.crt、dev/ca.key）
make dev-certs            # 等价：go run ./cmd/server gen-ca --out dev

# 2. 准备配置（数据库与 CA 指向 dev/）
cp config.example.yaml config.yaml
#   修改：database.path: dev/data.db
#        pki.ca_cert: dev/ca.crt
#        pki.ca_key:  dev/ca.key
#        listen / external_url 改成本机地址

# 3. 构建并运行
make web && make server
./bin/service-edge --config config.yaml
```

默认管理员账号由 `bootstrap_admin` 创建，访问 `http://<listen>/` 登录。

### 6.2 前端独立热更新开发

前端 dev server 已配置 `/api`、`/install`、`/download` 代理到 `http://127.0.0.1:18443`：

```bash
cd web && npm run dev      # http://127.0.0.1:5173
```

> 此模式下后端需放行该来源的 CORS（把 `http://127.0.0.1:5173` 加入 `cors.allowed_origins`）。

### 6.3 服务端命令与参数

```bash
service-edge --config config.yaml        # 启动控制面
service-edge --config config.yaml --agent-dist ./agent-dist   # 同时托管 Agent 二进制下载
service-edge gen-ca --out dev            # 生成自签 CA（仅开发/测试）
```

## 七、如何测试

### 7.1 单元测试

```bash
go test ./...        # 含 PKI 证书链校验、frp.toml 渲染等
go vet ./...
```

### 7.2 API 冒烟测试（手动）

启动后端后，可用 curl 走通核心链路（登录 → 建节点 → 安装命令 → Agent 注册 → 配置下发）：

```bash
B=http://127.0.0.1:18443
# 登录拿 JWT
TOKEN=$(curl -s -X POST $B/api/v1/auth/login -H 'Content-Type: application/json' \
  -d '{"username":"admin","password":"admin12345"}' | python3 -c 'import sys,json;print(json.load(sys.stdin)["token"])')
AUTH="Authorization: Bearer $TOKEN"
# 建 FRPS 节点
curl -s -X POST $B/api/v1/frps -H "$AUTH" -H 'Content-Type: application/json' \
  -d '{"name":"edge-1","bind_port":7000,"public_ip":"203.0.113.10"}'
```

完整脚本可参考 README 与本仓库提交记录中的冒烟流程。验证要点：
- 重复 `remote_port` 返回 `409`
- 安装 token 第一次注册 `200`、第二次 `403`（一次性）
- `GET /api/v1/agent/config` 在配置变更时**立即返回**新配置，无变更时 hang 约 30s 后 `304`

### 7.3 集成测试（需真实主机，design.md 阶段六）

1. 控制面建 FRPS → 在公网测试机执行安装命令 → 验证 frps 启动
2. 建 FRPC + 映射 → 在内网测试机执行安装命令 → 验证隧道连通
3. 修改配置 → 验证 long-polling 下发与热重载/重启
4. 制造配置错误 → 验证 Agent 回滚、上报 `ack(success=false)`、继续运行旧配置
5. 控制面下线 → 验证 Agent 离线降级（frp 进程不停、watchdog 保活）
6. 手动调短证书有效期 → 验证 ≤30 天自动续签下发

## 八、如何部署（生产）

### 8.1 部署控制面

```bash
# 1. 准备 CA（生产请用正式 PKI 签发，勿用 gen-ca）
mkdir -p /etc/service-edge /var/lib/service-edge /var/log/service-edge
cp ca.crt ca.key /etc/service-edge/
chmod 600 /etc/service-edge/ca.key

# 2. 放置二进制与配置
cp bin/service-edge /usr/local/bin/
cp config.yaml /etc/service-edge/config.yaml      # 改好密钥、域名、路径

# 3. 托管 Agent 二进制（供安装脚本下载）
mkdir -p /var/lib/service-edge/agent-dist
cp bin/agent_linux_amd64 /var/lib/service-edge/agent-dist/agent_linux_amd64
```

systemd unit（`/etc/systemd/system/service-edge.service`）：

```ini
[Unit]
Description=service-edge control plane
After=network.target

[Service]
ExecStart=/usr/local/bin/service-edge --config /etc/service-edge/config.yaml --agent-dist /var/lib/service-edge/agent-dist
Restart=on-failure
RestartSec=5s

[Install]
WantedBy=multi-user.target
```

```bash
systemctl daemon-reload && systemctl enable --now service-edge
```

### 8.2 反向代理与域名

- 前端 `edge.dreamreflex.com` 与后端 `edge-api.dreamreflex.com` 通过 Nginx/Caddy 转发到控制面。
- 后端 long-polling 接口会 hang 30s，反代需调大读超时（如 Nginx `proxy_read_timeout 60s;`），否则会被提前断开。
- 控制面默认监听 HTTP；TLS 终止可放在反代层，或在控制面前置负载均衡。

### 8.3 部署 Agent（frps / frpc）

在控制台创建节点后，复制生成的一次性安装命令，在目标机器以 root 执行：

```bash
curl -fsSL "https://edge-api.dreamreflex.com/install/frps.sh?token=XXXX" | sudo bash
```

脚本会：检测架构 → 下载 frp 与 Agent 二进制 → 写 `agent.yaml` 与 systemd unit → 向控制面注册 → 启动 Agent。Agent 随后 long-polling 拉取配置与证书，原子落盘并拉起 frp 进程。

目录布局：
- FRPS：`/opt/service-edge/frps-agent/`（单实例，unit `service-edge-frps`）
- FRPC：`/opt/service-edge/frpc-agent/instances/<uuid>/`（unit 模板 `service-edge-frpc@<uuid>`）

## 九、注意事项

### 9.1 安全

- **CA 私钥**是系统信任根，权限设为 `600`，控制面之外不外泄；本系统假设运行在受控可信环境。
- 生产务必修改 `bootstrap_admin` 默认密码，并使用足够随机的 `agent_api_token` / `jwt_secret`（`openssl rand -hex 32`）。
- 前后端分离，认证走 `Authorization: Bearer <jwt>`，**不使用 Cookie**；CORS 严格白名单，不开启 `Allow-Credentials`。
- 每个 frps 节点的 frp token 独立随机（64 位），单节点泄漏不影响其他节点。
- 一次性安装 token：15 分钟有效、单次消费（`UPDATE ... WHERE used_at IS NULL` 原子保证）。

### 9.2 启动强校验（不可绕过）

控制面启动时校验 CA：文件可读、cert/key 配对、在有效期内、具备 CA 能力。**任一失败直接 panic 退出**，不允许“降级模式”带病启动。若启动失败，先检查 CA 路径与证书有效性。

### 9.3 FRP 版本与配置语法

- 模板基于 **FRP v0.50+ 的 TOML 扁平语法**（`bindPort` / `auth.token` / `transport.tls.*` / `[[proxies]]`），**严禁使用旧 `.ini` / `bind_port` 旧风格**。
- 默认版本 `v0.61.1` 可在 `config.yaml` 或建节点表单覆盖；最新稳定版为 v0.68.x，语法兼容。
- TLS 自 v0.50 起默认开启；frpc 通过 `transport.tls.serverName = "frps-<uuid>"` 固定校验对端证书，与节点公网 IP 解耦。

### 9.4 网络与端口

- frps 的 `bind_port`（及可选 dashboard 端口、各 TCP/UDP `remote_port`）需在防火墙/安全组放行。
- 控制面 long-polling 需要反代放大读超时（见 8.2）。
- Agent 主机需能访问 GitHub（下载 frp）与控制面 `external_url`。内网受限环境可改 `frp_release.base_url` 指向私有镜像。

### 9.5 多实例限制

当前 Agent 为**单实例模型**：一个 Agent 进程管理一个 frp 实例（frps 节点或一个 frpc 实例）。同一主机部署多个 frpc 实例属集成测试阶段的增强项，目录布局已预留 `instances/<uuid>/`。

### 9.6 数据与回滚

- 证书、配置等文件类产物**全部存数据库**，不落控制面磁盘；操作回滚只需 SQL `ROLLBACK`，无磁盘残留。
- Agent 配置应用流程：暂存 `.new` → `frp verify` 校验 → 备份 → 原子 rename → reload/restart → 5s 存活检查；任一步失败则从备份回滚并保留旧配置运行。

### 9.7 可观测性

- 控制面日志走 `slog`（`LOG_LEVEL=debug` 提升级别）。
- 所有写操作（建/删/改 FRPS、FRPC、Proxy、生成安装命令、登录、配置应用结果）落审计日志，可在控制台“审计日志”页查询。

## 十、常见问题（FAQ）

| 现象 | 排查方向 |
|---|---|
| 控制面启动即 panic | CA 路径错误 / 证书过期 / cert 与 key 不配对 / 非 CA 证书 |
| 前端页面空白 | 未先 `make web` 再 `make server`，二进制内嵌的是占位前端 |
| 浏览器 CORS 报错 | 前端来源未加入 `cors.allowed_origins`（含协议与端口） |
| long-polling 频繁断开 | 反代 `proxy_read_timeout` 太短，调到 ≥60s |
| Agent 注册失败 403 | 安装 token 已过期或已被使用，重新生成安装命令 |
| Agent 配置应用失败 | 查看 Agent 日志与控制台审计（`config_apply_failed`）；多为 frp 配置语法或端口占用 |
| frp 下载失败 | Agent 主机无法访问 GitHub，改用私有镜像 `frp_release.base_url` |
| `remote_port` 创建报 409 | 该 frps 上端口已被占用，换端口或查“可用端口” |
